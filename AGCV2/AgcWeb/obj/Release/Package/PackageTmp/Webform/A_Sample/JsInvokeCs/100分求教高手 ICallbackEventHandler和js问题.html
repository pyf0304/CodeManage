解决方案 »

  1. 如果从客户端js传递到后台c#中的字符串中含有中文字符,则在回调之后显示乱码,或者不能成功回调,如何解决?
    ——————————————————————————————————————————
    ICallbackEventHandler可以访问页面上所有控件的Post值，你不需要使用eventArgument传递什么中文。从客户端GetCallbackEventReference()  到 客户端js函数接受回调后的返回值并显示的过程中,在后台的C#代码中不能发生任何与界面的交互吗?比如说 Response..
    ———————————————————————————————————————————
    GetCallbackResult本来就是一种Response.Write动作，只不过GetCallbackResult输出的东西可以被客户端端回调机制的javascript方法所解析，从而可以传递结果给你的javascript过程。
    事件回调过程中如果出现异常怎么调试?好象事件回调发生异常不报任何错误?郁闷
    ——————————————————————————————————————————
    你可以使用Try...Cache...捕获处理时期的异常，通过GetCallbackResult输出。
      

  2. 对上边中间那个问题，你可以在页面上设置一个List<string>类型的容器，每个需要写出内容的组件向这个容器里追加，然后在GetCallbackResult方法执行时从这个容器里一次性写出全部内容。
